// 001 · CONCEITO

Uma IA que
não esquece
e não vaza

Vella combina o OLLAMA — inferência de LLMs rodando silenciosa no seu hardware — com o Qdrant, banco vetorial que indexa e recupera embeddings por significado, não por palavras.

FastEmbed processa os modelos ONNX localmente. Nenhuma chamada externa. Nenhum dado em trânsito. O Delta Engine com XXHash3 regenera só o que mudou — 5.000+ arquivos em 0.3s.

O servidor MCP nativo expõe 11 ferramentas diretamente para Claude, Cursor e outros agentes via protocolo stdio.

📄 vella.config.toml
[core]
name      = "vella"
mode      = "local-only"
cloud     = false
telemetry = false

[ollama]
model  = "llama3.2"
embed  = "nomic-embed-text"
host   = "localhost:11434"

[qdrant]
host       = "localhost:6333"
collection = "vella_memory"
dims       = 768

[mcp]
transport = "stdio"
version   = "2024-11-05"

# seus dados. sua máquina. sempre.

// 003 · PIPELINE RAG

Como a Vella
processa tudo

INGESTÃO
Você alimenta
Documentos, código-fonte, PDFs, conversas. O PDR Generator varre o projeto e processa 5.000+ arquivos em menos de 0.3 segundos com análise semântica completa.
$ pdr gen --compress --auto-version
01
02
EMBEDDING LOCAL
FastEmbed converte
Cada fragmento é convertido em vetor de 768 dimensões pelo modelo all-MiniLM-L6-v2 rodando localmente via ONNX Runtime. Sem chamada de API externa.
model: all-MiniLM-L6-v2 · dims: 768
INDEXAÇÃO
Qdrant armazena
Os vetores são indexados no Qdrant com metadados ricos. Sessões, decisões e snapshots PDR são persistidos no banco Sled embutido na collection vella_memory.
collection: vella_memory · localhost:6333
03
04
BUSCA SEMÂNTICA
Similaridade de cosseno
Sua pergunta vira um vetor. O Qdrant encontra os fragmentos mais relevantes por similaridade de cosseno em milissegundos — não por palavras, por significado real.
$ pdr memory search "refatoração do motor"
RESPOSTA GERADA
OLLAMA responde
O modelo local recebe o contexto recuperado via RAG e gera a resposta. O servidor MCP entrega diretamente para Claude ou outro agente. Tudo local, tudo rápido.
$ pdr mcp # servidor MCP ativo via stdio
